API Endpoint Tasarımı

// apps/api/src/ai/ai.controller.ts POST /api/v1/ai/chat Body: { message: string, conversationId?: string } Auth: JWT (zorunlu) → AiService.chat(userId, message, conversationId) → Claude API (tool_use destekli) → { reply: string, conversationId: string, escalated?: boolean } GET /api/v1/ai/conversations → Kullanıcının geçmiş konuşmaları GET /api/v1/ai/conversations/:id/messages → Konuşma mesajları

Veritabanı Şeması

-- ai_conversations tablosu id uuid PK userId uuid FK → users source enum ('sahne' | 'mutfak' | 'mobile' | 'web') createdAt timestamp lastMessageAt timestamp -- ai_messages tablosu id uuid PK conversationId uuid FK → ai_conversations role enum ('user' | 'assistant' | 'tool') content text toolName varchar -- hangi tool çağrıldıysa createdAt timestamp -- feedbacks tablosuna eklenti source += 'gpt_escalation' -- AI'ın açtığı ticketlar

04

AI Core Engine — Detay

Multi-LLM Routing Mantığı

Her soru aynı modele gönderilmez. Görev tipine göre en uygun modele yönlendirilir.

Görev TipiModelNeden?
Genel soru, platform yönlendirme Claude Haiku Hızlı, düşük maliyet
Mevzuat analizi, SHKM süreçleri Claude Sonnet Güçlü reasoning
Karmaşık danışmanlık, raporlama Claude Opus En yüksek kalite
Hızlı yanıt, kısa soru GPT-4o mini Latency kritik durumlarda
Görsel / harita analizi GPT-4o Vision Görsel reasoning

Knowledge Graph (Qdrant)

Sistemin sektör hafızası. Her bilgi parçası vektör olarak saklanır; sorgu geldiğinde en alakalı içerikler Claude'a context olarak eklenir (RAG).

İndekslenen İçerikler

SHKM mevzuatı, LİHKAB süreçleri, sık sorulan sorular, eğitim materyalleri, etkinlik açıklamaları, çözüme kavuşmuş ticket geçmişleri

Güncelleme Sıklığı

Yeni mevzuat eklendiğinde otomatik; ticket çözüldüğünde öğrenme döngüsüyle; admin manuel içerik ekleyebilir

07

Çözüm Arıyorum ↔ HaritailesiGPT Entegrasyonu

İki sistem özerk görünür ama tek altyapıya bağlıdır. Kullanıcı Sahne/Mutfak'ta bir konuşma başlatır, telefonda devam edebilir. Konuşma geçmişi ai_conversations tablosunda kayıtlıdır.

// Konuşma kaynak takibi ai_conversations.source = 'sahne' | 'mutfak' | 'mobile' | 'web' // Escalation kaynağı — admin bu farkı görmeli feedbacks.source = 'gpt_escalation' feedbacks.metadata = { conversationId: "uuid", lastMessages: [...], // son 3 mesaj agentUsed: "regulation_agent", confidence: 0.32 // AI'ın çözüm güveni düşük → ticket }

Escalation Kuralları

DurumEylem
AI confidence skoru < 0.4 Otomatik ticket aç, uzman yönlendir
Kullanıcı "insan ile konuşmak istiyorum" der Anında ticket aç
Aynı soru 3 kez tekrar edilir Ticket aç + admin bildir
Hukuki/mevzuat sorusu (yüksek risk) Yanıt ver + "uzman onaylasın" ticket aç

Geliştirme Yol Haritası

⚠️ Öneri: HaritailesiGPT mobil uygulama geliştirmesine başlamadan önce Sahne, Mutfak ve vakıf web sitesi tamamlanmalıdır. AI Engine ise her iki süreçle paralel ilerleyebilir.

AI Core Engine (API Katmanı)

apps/api/src/ai/ Claude API entegrasyonu Tool calling ai_conversations tablosu
